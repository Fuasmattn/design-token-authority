# TICKET-028: `DESIGN.md` generator for AI coding agents

**Phase:** 7 — AI Interop
**Priority:** High
**Effort:** M

## Summary

Add a `dta design-md` command that generates a `DESIGN.md` (and/or `AGENTS.md`-compatible variants) at the project root, giving AI coding agents — Claude Code, Cursor, v0, Codex, Copilot Workspace — a compact, machine-readable reference to the design system. This is the AI-facing counterpart to the human-facing `dta docs` site (TICKET-022).

## Background

AI coding agents now routinely read project-level context files (`CLAUDE.md`, `AGENTS.md`, `.cursor/rules`, `.github/copilot-instructions.md`) before generating code. In projects with design tokens, these files are usually silent about tokens — so the agent hardcodes `#003F8A`, invents Tailwind classes that don't exist, or ignores brand switching altogether.

A generated `DESIGN.md` fixes this at the source: the token pipeline already knows every token, alias chain, brand, and breakpoint; emitting that knowledge in a format agents consume turns the whole token system into "ambient context" for every code generation.

Unlike `dta docs` (which optimizes for browsing by humans), `DESIGN.md` optimizes for:

- **Small token budget** — dense, minimal prose, flat token tables
- **Unambiguous naming** — the exact identifier an agent should type in code
- **Usage rules** — "do / don't" patterns the agent should follow
- **Framework-aware import hints** — where to pull a token from, per target

## Acceptance Criteria

- [ ] New command: `dta design-md` (writes to project root by default; `--out <path>` configurable)
- [ ] Output is a single markdown file under a typical budget (<2000 tokens when rendered)
- [ ] Sections:
  - **Overview** — project name, brands, breakpoints, output targets
  - **Token reference** — flat tables grouped by category (color, spacing, typography, effects), each row: `semantic name`, `resolved value`, `alias chain`, `code identifier per target`
  - **Brand switching** — how brands are selected in each output target (CSS class, Tailwind class, Swift struct, etc.)
  - **Usage rules** — "do" and "don't" bullets derived from config (`lint.rules`) plus project-wide invariants
  - **Do not invent tokens** — explicit instruction + reference to the canonical token file location
- [ ] `--format agents-md` writes `AGENTS.md` alongside or instead of `DESIGN.md` (identical content, different filename for tools that key off `AGENTS.md`)
- [ ] `--format compact` emits a minimal, ~500-token summary for tight context windows
- [ ] Runs as part of `dta build` when `designMd: true` is set in config
- [ ] Regenerating is idempotent — the file has a header warning against manual edits
- [ ] Multi-brand: one table per brand where brand-scoped tokens diverge

## Implementation Notes

**File header (generated):**

```markdown
<!-- AUTOGENERATED by design-token-authority. Do not edit by hand. -->
<!-- Regenerate with: npx dta design-md -->

# Design System Reference

This file is context for AI coding assistants. It describes the design tokens
available in this project and the rules for using them in code.
```

**Token table row example:**

```markdown
| Semantic | Value | Alias chain | CSS | Tailwind | Swift |
|---|---|---|---|---|---|
| Brand primary | `#003F8A` | `Colors.Text.Default → Colors.Brand.BrandA.Primary → Colors.Foundation.Blue.700` | `var(--color-brand-primary)` | `text-brand-primary` | `BrandTheme.BrandA.primary` |
```

**Rule derivation:** Reuse `lint.rules` from TICKET-018 — e.g. `semantic-must-alias` becomes "Don't hardcode hex colors in components; use the semantic token." Project config can add custom rules in a `designMd.rules` array.

**Target-aware identifiers:** Already produced by formatters (TICKET-009/010/011/012/013). This command pulls from build output rather than recomputing.

**Placement strategy:** `DESIGN.md` at project root is read by Claude Code automatically when adjacent to a `CLAUDE.md`. For tools keyed to `AGENTS.md`, emit both (same content). Add a linked reference from `CLAUDE.md` so human readers know it's autogenerated.

**Compact format:** A grouped list rather than a table, with only semantic name + code identifier per target — no values, no alias chains. For cases where the full reference blows the context window.
